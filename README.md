# 👑 The System Design Conquest: Guild Master Guide

Welcome to the ultimate System Design Conquest repository. This serves as our group's single source of truth, central library, and ledger as we scale our engineering mindsets from SDE-1 up to Senior Software Engineer (L6) scale.

---

## 🎯 Conquest Overview
The objective of this campaign is to master the physics of distributed systems, data-intensive architectures, and high-availability design. We are pairing the theory of Martin Kleppmann’s **Designing Data-Intensive Applications (DDIA)** with real-world architectural blueprints from modern tech titans.

---

## 📜 The Guild Commandments & Conquest Rules

1. **The Proof-of-Work Rule:** No XP is official until the corresponding markdown notes, architecture diagrams, or functional code is committed to this repository via a Pull Request.
2. **The Weekly Ritual (PR Check-in):** Every Sunday, each player must submit or update a PR logging their completed tasks for the week, linking their artifacts, and updating their line on `LEADERBOARD.md`.
3. **The Mentor Bonus:** If a player helps unblock a teammate on a complex topic (e.g., explaining isolation levels or Paxos consensus) via a 15-minute whiteboard session, they earn **+15 Mentor XP** upon PR verification.
4. **The Dead Weight Penalty:** If a player goes **7 consecutive days** without earning any XP, they lose 15 XP from their total pool. If they are at 0 XP, they owe the group one cup of coffee/drink at the next team sync.
